# 01_requirements.md - 邮件提取客户端需求规格

## 1. 项目概述

开发一个本地 Python 客户端工具，用于从 Outlook 提取邮件并提交给后端处理。后端提供异步任务处理能力。

## 2. 用户故事

### 2.1 主界面
- **作为**用户，我希望打开客户端时看到两个按钮（邮件提取、待规划）
- **因为**待规划功能尚未开发，该按钮应为灰色不可点击状态

### 2.2 邮件提取功能
- **作为**用户，我希望能够指定日期区间来筛选邮件
- **因为**我只需要特定时间范围内的邮件

- **作为**用户，我希望能够选择特定的邮件文件夹
- **因为**收件箱不是唯一的邮件来源

- **作为**用户，我希望通过关键词搜索来筛选邮件
- **因为**我需要快速定位特定主题的邮件

- **作为**用户，我希望同一 ConversationTopic 只保留最新的一封
- **因为**避免重复提取同一会话的多封邮件

- **作为**用户，我希望能够多选邮件后批量提取
- **因为**提高操作效率

- **作为**用户，我希望将选中的邮件以 Word 文档形式发送给后端
- **因为**后端需要 Word 格式进行处理（Word 文件名=ConversationTopic，内容=最后一封邮件的正文，含图片）

### 2.3 任务状态查询
- **作为**用户，我希望提交后能获得 task_id
- **因为**用于后续查询任务状态

- **作为**用户，我希望能够查询任务当前状态
- **因为**了解处理进度

- **任务状态包括**：未提取、进行中、完成、失败

## 3. 功能需求

### 3.1 前端（Python 客户端）

| 功能 | 描述 |
|------|------|
| 主界面 | 显示"邮件提取"和"待规划"按钮，待规划按钮灰色不可点击 |
| 邮件提取窗口 | 新窗口显示邮件列表和筛选条件 |
| 日期区间筛选 | 支持设置开始日期和结束日期（闭区间） |
| 文件夹选择 | 树形结构显示 Outlook 文件夹，支持选择任意文件夹，默认选择收件箱 |
| 关键词搜索 | 支持通过关键词搜索邮件主题 |
| 去重逻辑 | 同一 ConversationTopic 只保留最新的一封 |
| 邮件多选 | 支持勾选多封邮件 |
| 提取提交 | 将选中邮件转换为 Word 文档并 POST 到后端（每个邮件一个 Word，文件名=ConversationTopic，内容=最后一封邮件的正文，含图片），多次 POST，每次显示进度条 |
| 状态查询 | 用户主动点击按钮查询任务状态 |

### 3.2 后端（FastAPI）

| 接口 | 方法 | 描述 |
|------|------|------|
| `/upload` | POST | 接收 Word 文件，返回 task_id |
| `/task/{task_id}` | GET | 查询任务状态 |

### 3.3 数据结构

**任务状态枚举**：
- `pending` - 未提取
- `processing` - 进行中
- `completed` - 完成
- `failed` - 失败（不返回错误信息，引导用户重试）

**后端地址**：http://localhost:8000

## 4. 验收标准

1. 主界面正确显示两个按钮，待规划按钮灰色不可点击
2. 点击邮件提取按钮打开新窗口
3. 日期区间筛选功能正常工作
4. 文件夹选择功能正常工作
5. 关键词搜索功能正常工作
6. 同一 ConversationTopic 只保留最新邮件
7. 用户可以多选邮件并提交
8. 提交后返回 task_id
9. 前端可以通过主动点击查询任务状态
10. 后端接口返回正确的状态枚举值
